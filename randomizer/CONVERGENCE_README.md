# 模型推理收敛过程模拟 - 使用说明

## 概述

这个脚本模拟了模型从完全随机逐步收敛到正确输出的10次推理过程。每次推理都会逐步减少随机化的范围，增加保留正确值的比例，最终接近正确输出。

## 收敛策略

### 10次推理的详细策略

| 推理次数 | 描述 | 随机化比例 | 保留正确值比例 | 准确率 |
|---------|------|-----------|--------------|--------|
| 第1次 | 完全随机 | 100.0% | 0.0% | 0.0% |
| 第2次 | 保留开头和结尾各10% | 80.2% | 19.8% | 19.8% |
| 第3次 | 保留开头和结尾各20% | 59.9% | 40.1% | 40.1% |
| 第4次 | 保留开头和结尾各30% | 40.1% | 59.9% | 59.9% |
| 第5次 | 保留开头和结尾各35% | 30.2% | 69.8% | 69.8% |
| 第6次 | 随机分散的4个区域 | 26.0% | 74.0% | 74.0% |
| 第7次 | 随机3个较小区域 | 15.6% | 84.4% | 84.4% |
| 第8次 | 随机2个小区域 | 7.8% | 92.2% | 92.2% |
| 第9次 | 随机1个很小区域 | 2.6% | 97.4% | 97.4% |
| 第10次 | 几乎完全正确 | 1.3% | 98.7% | 98.7% |

## 使用方法

### 1. 生成收敛数据

```bash
# 为单个文件生成10次推理的收敛数据
python3 simulate_convergence.py \
    data_test/eval_traj_100.json \
    convergence_output_100/ \
    --seed 42
```

### 2. 验证收敛数据

```bash
# 验证生成的数据是否正确
python3 verify_convergence.py \
    data_test/eval_traj_100.json \
    convergence_output_100/
```

### 3. 查看收敛策略

```bash
# 查看详细的收敛策略说明
cat convergence_output_100/convergence_strategy.txt
```

## 收敛策略详解

### 阶段1：初始随机阶段（第1-2次）
- **第1次**：完全随机，模拟模型初始状态
- **第2次**：开始保留开头和结尾的特征，中间部分仍然随机

### 阶段2：快速收敛阶段（第3-5次）
- 从两端向中间逐步收敛
- 每次推理保留更多的正确值
- 准确率从20%快速提升到70%

### 阶段3：精细调整阶段（第6-8次）
- 从连续区域转变为分散区域
- 模拟模型对不同特征区域的精细调整
- 准确率从74%提升到92%

### 阶段4：最终收敛阶段（第9-10次）
- 只随机化极少数位置
- 模拟模型接近最优解
- 准确率达到97%-99%

## 实际应用场景

### 场景1：训练过程可视化
使用这10组数据可以展示模型训练过程中的输出变化，帮助理解模型如何从随机状态逐步学习到正确的输出模式。

### 场景2：模型评估
可以在不同的收敛阶段评估模型性能，了解模型在不同准确率下的表现。

### 场景3：数据增强
这10组数据可以作为数据增强的样本，提供不同噪声水平的训练数据。

### 场景4：渐进式学习
模拟渐进式学习过程，从简单到复杂逐步训练模型。

## 自定义收敛策略

如果需要自定义收敛策略，可以修改 `simulate_convergence.py` 中的 `design_convergence_strategy()` 函数。

示例：修改为更激进的收敛策略（更快收敛）

```python
def design_convergence_strategy():
    strategies = []

    # 第1次：完全随机
    strategies.append({
        'iteration': 1,
        'description': '完全随机',
        'positions': set(range(0, 384))
    })

    # 第2次：只随机50%
    strategies.append({
        'iteration': 2,
        'description': '随机50%',
        'positions': set(range(96, 288))
    })

    # ... 继续定义其他策略
    return strategies
```

## 注意事项

1. **随机种子**：使用相同的随机种子可以保证结果可重复
2. **数据一致性**：所有推理都基于同一个正确输出文件
3. **Token数量**：确保所有生成的数据都保持384个token
4. **格式正确性**：生成的数据格式与原始数据完全一致
